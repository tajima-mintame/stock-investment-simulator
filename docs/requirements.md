# 要件定義書: 株式投資シミュレーター

## 1. プロジェクト概要

### 1.1 目的

株式投資の仮想売買を行うシミュレーターを構築する。実際の株価データを用いて、リスクなく投資戦略の検証・学習を行える環境を提供する。

### 1.2 背景

- 実際の資金を使わずに投資戦略を試行錯誤したい
- 株価データの蓄積と分析を通じて市場の傾向を理解したい
- テクニカル分析やポートフォリオ理論を実データで学習したい

### 1.3 対象ユーザー

個人利用（シングルユーザー）。投資の学習・検証を目的とする開発者。

## 2. 対象市場

### 2.1 初期リリース対象

| 市場 | 取引所 | データソース | 通貨 |
|------|--------|-------------|------|
| 日本株 | 東京証券取引所（TSE） | J-Quants API V2（JPX公式） | JPY |

### 2.2 将来対応予定

| 市場 | 取引所 | データソース | 通貨 |
|------|--------|-------------|------|
| 米国株 | NYSE / NASDAQ | Finnhub API（無料枠: 60 req/min） | USD |

プロバイダーの抽象インターフェース（`MarketDataProvider`）とDBスキーマの `market` カラムは、将来の米国株対応を見据えて設計済み。

### 2.3 データソースの制約

| API | 無料枠の制限 |
|-----|-------------|
| J-Quants V2 | APIキー認証。個人向け無料プランあり |

## 3. 機能要件

### 3.1 株価データ管理

| ID | 要件 | 詳細 |
|----|------|------|
| F-DATA-01 | 銘柄情報の取得 | 銘柄コード、企業名、セクター、通貨を外部APIから取得しDBに保存する |
| F-DATA-02 | 日足データの取得 | 指定期間のOHLCV（始値・高値・安値・終値・出来高）を取得しDBに保存する |
| F-DATA-03 | 手動同期 | ユーザーが銘柄・期間を指定してデータ取得をトリガーできる |
| F-DATA-04 | 自動収集 | バックグラウンドで定期的にデータを収集する（平日16:00 JST、TSE閉場後） |
| F-DATA-05 | 銘柄検索 | 銘柄コードまたは企業名で外部APIを検索できる |
| F-DATA-06 | 収集ログ | データ取得の成否をログテーブルに記録する |

### 3.2 仮想売買

| ID | 要件 | 詳細 |
|----|------|------|
| F-TRADE-01 | 仮想口座 | 初期資金10万円（JPY）の仮想口座を持つ |
| F-TRADE-02 | 買い注文 | 銘柄・数量・価格を指定して仮想的に買い注文を実行する。残高から代金を差し引く |
| F-TRADE-03 | 売り注文 | 保有銘柄を指定して仮想的に売り注文を実行する。残高に代金を加算する |
| F-TRADE-04 | 残高管理 | 現金残高とポートフォリオ時価総額を常に把握できる |
| F-TRADE-05 | 取引履歴 | 全取引のログ（銘柄、売買区分、数量、価格、日時）を記録・閲覧できる |
| F-TRADE-06 | 損益統計 | 総リターン、勝率、平均利益、平均損失を算出する |

### 3.3 テクニカル分析

| ID | 要件 | 詳細 |
|----|------|------|
| F-IND-01 | 移動平均線（MA） | 任意期間（5, 25, 75日など）の単純移動平均線を算出・チャートにオーバーレイする |
| F-IND-02 | RSI | 14日RSI（Wilder平滑化）を算出・サブチャートに表示する |
| F-IND-03 | MACD | MACD線（12,26）、シグナル線（9）、ヒストグラムを算出・サブチャートに表示する |
| F-IND-04 | ボリンジャーバンド | 20日移動平均 ± 2σのバンドを算出・チャートにオーバーレイする |

### 3.4 銘柄スクリーニング

| ID | 要件 | 詳細 |
|----|------|------|
| F-SCR-01 | 出来高フィルタ | 平均出来高の下限・上限で絞り込む |
| F-SCR-02 | ボラティリティフィルタ | 日次リターンの標準偏差で絞り込む |
| F-SCR-03 | セクターフィルタ | セクターで絞り込む |
| F-SCR-04 | ソート | 出来高、ボラティリティ、変動率などでソートする |

### 3.5 ポートフォリオ分析

| ID | 要件 | 詳細 |
|----|------|------|
| F-PORT-01 | 保有一覧 | 保有銘柄の数量、平均取得単価、現在価格、含み損益を一覧表示する |
| F-PORT-02 | セクター配分 | 保有銘柄のセクター別の配分比率を表示する |
| F-PORT-03 | 相関分析 | 保有銘柄間の日次リターンのピアソン相関行列を算出・表示する |

## 4. 非機能要件

| ID | カテゴリ | 要件 |
|----|---------|------|
| NF-01 | 利用形態 | ローカル環境で動作するWebアプリケーション |
| NF-02 | ユーザー数 | シングルユーザー（認証不要） |
| NF-03 | データ粒度 | 日足（分足・ティックは対象外） |
| NF-04 | レート制限 | 外部APIのレート制限を遵守する仕組みを組み込む |
| NF-05 | セキュリティ | APIキーを環境変数で管理し、コードにハードコードしない |
| NF-06 | セキュリティ | SQLインジェクション防止のためパラメータ化クエリを使用する |
| NF-07 | データ永続化 | SQLiteファイルとしてローカルに保存する |
| NF-08 | ブラウザ対応 | モダンブラウザ（Chrome, Firefox, Safari）で動作する |

## 5. スコープ外

以下は本プロジェクトのスコープに含めない。

- マルチユーザー対応（認証・認可）
- リアルタイム株価配信（WebSocket）
- 自動売買アルゴリズムの実行
- 本番環境へのデプロイ・Docker化
- 分足・ティックデータの取得
- 信用取引・空売りのシミュレーション
- 米国株対応（将来フェーズで対応予定）
- 為替変換

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| OHLCV | Open（始値）, High（高値）, Low（安値）, Close（終値）, Volume（出来高）の5つの日足データ |
| MA | Moving Average（移動平均線） |
| RSI | Relative Strength Index（相対力指数）。0〜100の範囲で買われすぎ・売られすぎを示す |
| MACD | Moving Average Convergence Divergence（移動平均収束拡散法） |
| ボリンジャーバンド | 移動平均線を中心に標準偏差で上下のバンドを描画するテクニカル指標 |
| J-Quants | 日本取引所グループ（JPX）が提供する株式データAPI |
| Finnhub | 米国株式データを提供するAPIサービス（将来対応予定） |
